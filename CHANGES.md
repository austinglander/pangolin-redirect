## AI Generated Summary of Changes

Adds a new "redirect" resource type that returns an HTTP redirect (301/302) to a fixed admin-configured URL instead of proxying to a backend service.

### What changed

**Schema** (`server/db/sqlite/schema/schema.ts`, `server/db/pg/schema/schema.ts`)
- Added columns to `resources`: `type`, `redirectUrl`, `preservePath`, `redirectCode`
- Migration is auto-generated at build time by `npm run db:generate`

**API**
- `server/routers/resource/createResource.ts` — New `createRedirectResource` handler and Zod schema
- `server/routers/resource/updateResource.ts` — New `updateRedirectResource` handler and Zod schema
- `server/routers/resource/listResources.ts` — Added `type` and `redirectUrl` to response
- `server/routers/target/createTarget.ts` — Blocks target creation on redirect resources

**Traefik config** (`server/lib/traefik/getTraefikConfig.ts`)
- Separate query for redirect resources (no targets/sites dependency)
- Generates `redirectRegex` middleware, dummy service, and optional path preservation
- Proxy query excludes redirect resources to prevent conflicts

**Frontend**
- `src/app/[orgId]/settings/resources/proxy/create/page.tsx` — Redirect type in creation wizard
- `src/app/[orgId]/settings/resources/proxy/[niceId]/general/page.tsx` — Redirect fields in settings
- `src/app/[orgId]/settings/resources/proxy/[niceId]/layout.tsx` — Proxy tab hidden for redirects
- `src/app/[orgId]/settings/resources/proxy/[niceId]/proxy/page.tsx` — Redirects to general tab
- `src/components/ProxyResourcesTable.tsx` — Type-aware protocol column, status, and links
- `src/components/ResourceInfoBox.tsx` — Shows redirect destination
- `messages/en-US.json` — i18n keys for redirect UI

### How it works

- Redirect resources have `type="redirect"`, `http=true`, `ssl=true`, and no targets
- Traefik serves the redirect via `redirectRegex` middleware with a dummy backend service
- Auth (badger middleware) runs before the redirect, so SSO/pincode/password protection works
- `preservePath=true` appends the original path and query string to the redirect URL
- Existing resources are unaffected (default `type="proxy"`)
